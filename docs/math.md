# 四旋翼无人机数学基础

四旋翼无人机的控制、导航与状态估计都建立在一套稳定的数学工具之上。本页从工程视角梳理几类最常见的基础知识，帮助你理解姿态控制、位置控制和传感器融合背后的核心思路。

## 为什么四旋翼需要数学模型

四旋翼并不是“直接给电机油门就能稳定飞行”的系统。飞控在每一个控制周期都需要回答几个问题：

- 当前机体的姿态和位置是什么。
- 目标姿态、目标速度或目标位置是什么。
- 当前状态与目标状态之间误差有多大。
- 应该如何分配四个电机的推力来修正误差。

这些问题分别对应到线性代数、坐标变换、刚体姿态表示、动力学建模以及状态估计。

## 线性代数基础

线性代数是四旋翼建模最常用的语言。状态量、控制量和传感器测量值几乎都可以写成向量或矩阵。

### 向量与状态表示

在飞控系统中，常见状态向量可以写成：

```text
x = [p_x, p_y, p_z, v_x, v_y, v_z, phi, theta, psi]^T
```

其中：

- `p` 表示位置。
- `v` 表示速度。
- `phi, theta, psi` 分别表示滚转角、俯仰角、偏航角。

如果系统使用四元数，也常写成：

```text
x = [p, v, q, omega]^T
```

这里：

- `p` 是位置向量。
- `v` 是速度向量。
- `q` 是姿态四元数。
- `omega` 是角速度向量。

### 矩阵与线性变换

矩阵常用于：

- 坐标系之间的旋转。
- 线性系统近似。
- 多旋翼控制分配。
- 卡尔曼滤波中的协方差传播。

例如三维向量 `v` 经过旋转矩阵 `R` 后可写成：

```text
v_world = R * v_body
```

这表示把机体系中的向量转换到世界坐标系中。

### 特征值与系统稳定性

在线性控制理论里，系统常被写为：

```text
x_dot = A x + B u
```

矩阵 `A` 的特征值决定了系统局部稳定性：

- 特征值实部小于 0，系统趋于稳定。
- 特征值实部大于 0，系统会发散。

虽然实际四旋翼是非线性系统，但在平衡点附近常会线性化，便于分析控制器性能。

## 坐标系与坐标转换

四旋翼控制中最容易混淆的内容之一就是坐标系。只要坐标系定义不清，控制方向就很容易写反。

### 常见坐标系

通常会涉及两类坐标系：

1. 世界坐标系
2. 机体坐标系

世界坐标系一般固定在地面，常用于表示位置、航点和全局速度；机体坐标系固定在无人机机身上，随姿态变化而变化。

常见约定如下：

- 世界系 `W`：`x` 指向前方或东，`y` 指向左方或北，`z` 指向上方。
- 机体系 `B`：`x` 指向机头，`y` 指向左翼，`z` 指向上方。

不同飞控或论文中可能使用 `NED` 或 `ENU` 坐标系，因此实现时必须确认符号约定。

### 旋转矩阵

如果四旋翼姿态由旋转矩阵 `R` 表示，则 `R` 满足：

```text
R^T R = I
det(R) = 1
```

这表示：

- 列向量互相正交。
- 旋转不会改变向量长度。
- 旋转不包含镜像变换。

把机体系加速度转换到世界系时，常写成：

```text
a_world = R * a_body
```

而从世界系转换回机体系则常用：

```text
a_body = R^T * a_world
```

### 欧拉角与分步旋转

很多飞控实现会用滚转 `phi`、俯仰 `theta`、偏航 `psi` 表示姿态。若按 `Z-Y-X` 顺序旋转，则总旋转矩阵可以写成：

```text
R = R_z(psi) R_y(theta) R_x(phi)
```

它的优点是直观，便于理解“先偏航、再俯仰、再滚转”的过程；缺点是在某些姿态附近会出现奇异性问题。

## 欧拉角与四元数

四旋翼姿态表示最常见的两种方法是欧拉角和四元数。

### 欧拉角

欧拉角的优点：

- 物理意义直观。
- 便于人工调试和可视化。
- 控制误差容易解释。

欧拉角的缺点：

- 存在万向节锁问题。
- 大角度机动时数值表现较差。
- 角速度与姿态角变化率关系并不简单。

例如角速度向量 `omega = [p, q, r]^T` 与欧拉角导数 `[\dot phi, \dot theta, \dot psi]^T` 并不是简单相等关系，而是通过一个姿态相关矩阵耦合起来。

### 四元数

四元数常写为：

```text
q = [q_w, q_x, q_y, q_z]^T
```

它可以看作“旋转轴 + 旋转角”的紧凑表示方式。若绕单位轴 `u = [u_x, u_y, u_z]^T` 旋转角度 `theta`，则四元数为：

```text
q = [cos(theta/2), u_x sin(theta/2), u_y sin(theta/2), u_z sin(theta/2)]^T
```

四元数的优势：

- 没有万向节锁。
- 更适合连续积分和姿态插值。
- 在姿态估计与滤波中更稳定。

工程上需要注意：

- 单位四元数必须满足 `||q|| = 1`。
- 每次积分更新后通常都要重新归一化。
- `q` 和 `-q` 表示同一个物理姿态。

### 四元数到旋转矩阵

设四元数为 `q = [w, x, y, z]^T`，对应旋转矩阵可写为：

```text
R(q) =
[1 - 2(y^2 + z^2), 2(xy - wz),     2(xz + wy)
 2(xy + wz),       1 - 2(x^2 + z^2), 2(yz - wx)
 2(xz - wy),       2(yz + wx),     1 - 2(x^2 + y^2)]
```

因此四元数最终仍然能转换成旋转矩阵，用于坐标变换和推力方向计算。

## 四旋翼动力学的基本形式

四旋翼本质上是一个六自由度刚体系统，包含三维平动和三维转动。

### 平动方程

设四个桨产生的总推力为 `T`，质量为 `m`，重力加速度为 `g`，则平动方程常写成：

```text
m p_ddot = R * [0, 0, T]^T + [0, 0, -mg]^T
```

含义是：

- 推力方向由机体 `z` 轴经过姿态矩阵 `R` 映射到世界系。
- 重力始终作用在世界坐标系竖直方向。

这也是为什么姿态控制会直接影响位置控制。无人机想向前飞，并不是“产生前向推力”，而是先俯仰，让总推力向前倾斜。

### 转动方程

转动动力学常由刚体欧拉方程描述：

```text
J omega_dot + omega x (J omega) = tau
```

其中：

- `J` 是惯性矩阵。
- `omega` 是机体角速度。
- `tau` 是总控制力矩。

三个轴上的控制力矩通常来自：

- 电机推力差引起的滚转力矩。
- 电机推力差引起的俯仰力矩。
- 螺旋桨反扭矩引起的偏航力矩。

## 控制分配的数学关系

四旋翼飞控最终要把“总推力 + 三轴力矩”的控制需求分配给四个电机。

若电机推力为 `f_1, f_2, f_3, f_4`，则常见的控制分配写法是：

```text
[ T   ]   [ 1  1  1  1 ] [f_1]
[tau_x] = [ 0  l  0 -l ] [f_2]
[tau_y]   [-l  0  l  0 ] [f_3]
[tau_z]   [ c -c  c -c ] [f_4]
```

其中：

- `l` 是电机到机体中心的力臂长度。
- `c` 是反扭矩系数。

这本质上是一个矩阵映射问题。已知期望控制量 `[T, tau_x, tau_y, tau_z]^T` 后，就可以通过求逆或伪逆计算四个电机推力指令。

## 卡尔曼滤波

真实飞行时，无人机无法直接获得“完美状态”，只能通过 IMU、气压计、GPS、磁力计、视觉里程计等传感器进行估计。卡尔曼滤波就是最常见的状态估计工具之一。

### 为什么需要滤波

不同传感器各有优缺点：

- 陀螺仪短时间响应快，但积分后会漂移。
- 加速度计能感知重力方向，但受机动加速度干扰较大。
- GPS 能给全局位置，但频率低、噪声大。
- 气压计能估计高度，但容易受气流影响。

因此需要一种方法，把系统模型和多个传感器测量结合起来，得到更稳定的状态估计。

### 线性卡尔曼滤波

线性离散系统通常写成：

```text
x_k = A x_{k-1} + B u_k + w_k
y_k = H x_k + v_k
```

其中：

- `x_k` 是时刻 `k` 的状态。
- `u_k` 是控制输入。
- `y_k` 是测量值。
- `w_k` 是过程噪声。
- `v_k` 是测量噪声。

卡尔曼滤波分两步：

1. 预测：利用系统模型预测当前状态。
2. 更新：利用传感器观测修正预测结果。

核心增益为：

```text
K_k = P_k^- H^T (H P_k^- H^T + R)^-1
```

其中：

- `P_k^-` 是预测协方差。
- `R` 是测量噪声协方差。
- `K_k` 越大，表示越相信测量值。

### 扩展卡尔曼滤波

四旋翼姿态和运动方程通常是非线性的，因此工程上更常见的是扩展卡尔曼滤波 `EKF`。

EKF 的思路是：

- 使用非线性函数描述系统状态传播和测量过程。
- 在当前工作点对非线性函数做一阶线性化。
- 使用雅可比矩阵代替线性卡尔曼滤波中的 `A` 和 `H`。

在四旋翼中，EKF 常用于估计：

- 姿态四元数。
- 陀螺仪零偏。
- 速度与位置。
- 高度或视觉定位状态。

### 一个典型的姿态融合思路

以 IMU 姿态估计为例：

- 陀螺仪提供角速度，用于短时积分预测姿态。
- 加速度计测得重力方向，用于修正俯仰和滚转。
- 磁力计提供地磁方向，用于修正偏航。

这样可以兼顾响应速度和长期稳定性。

## 从数学到飞控实现

如果把四旋翼飞控系统拆开看，常会得到下面这条链路：

1. 使用四元数或旋转矩阵表示姿态。
2. 使用坐标变换把传感器数据统一到同一坐标系。
3. 使用卡尔曼滤波或互补滤波估计状态。
4. 基于动力学模型设计姿态环、角速度环和位置环控制器。
5. 通过控制分配矩阵把控制量转换成四个电机输出。

这说明数学并不是独立的理论模块，而是从传感器输入一路贯穿到电机输出。

## 学习建议

如果你准备进一步阅读飞控代码或自行实现控制算法，建议按下面顺序掌握：

1. 三维向量、矩阵乘法、转置与逆矩阵。
2. 欧拉角、旋转矩阵、四元数之间的转换。
3. 刚体平动与转动动力学。
4. PID 控制与线性化分析。
5. 卡尔曼滤波、扩展卡尔曼滤波和传感器融合。

当这些概念串起来后，你会更容易理解 PX4、ArduPilot 或自研飞控中的姿态解算、状态估计和控制分配模块。
